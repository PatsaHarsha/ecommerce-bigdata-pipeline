4.2 Transformation vs Action in Spark
A transformation (e.g., df.filter(col("salary") > 70000)) is lazy — it builds a
logical plan but executes nothing. An action (e.g., df.count()) triggers Spark to
actually compute the DAG. In Part 3 of the notebook, calling tanzanians.show() was
the action that caused Spark to scan the data, apply the filter transformation, and
return results.

4.3 Why does Delta Lake store a delta_log/ folder?
The delta_log/ folder is Delta Lake's transaction log — an append-only journal of
every write operation. Each commit produces a new JSON file (0000000000000000000N.json)
recording what changed, when, and with what schema. This enables ACID guarantees,
full audit history, and "time travel" (reading data at any past version). Without
the log, Delta would just be ordinary Parquet with no rollback capability.
